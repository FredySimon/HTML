Formularios

El form elemento HTML define un formulario que se utiliza para recopilar la entrada del usuario. Los elementos de formulario son diferentes tipos de elementos de entrada, como campos de texto, casillas de verificación, botones de opción, botones de envío y más.

El input elemento es el elemento de forma más importante. EL elemento se puede mostrar de varias maneras, dependiendo del atributo de tipo.

El controlador de formulario se especifica en el atributo de acción del formulario. El action atributo define la acción a realizar cuando se envía el formulario.

Si action se omite el atributo, la acción se establece en la página actual.

El target atributo especifica si el resultado enviado se abrirá en una nueva pestaña del navegador, un marco o en la ventana actual.

Otros valores legales son "_parent", "_top" o un nombre que representa el nombre de un iframe.

El method atributo especifica el método HTTP ( GET o POST ) que se utilizará al enviar los datos del formulario.

El método predeterminado al enviar datos del formulario es GET. Sin embargo, cuando se utiliza GET, los datos del formulario enviado serán visibles en el campo de dirección de la página.

  • Agrega datos de formulario a la URL en pares de nombre / valor.
  • La longitud de una URL es limitada (2048 caracteres).
  • ¡Nunca use GET para enviar datos confidenciales! (será visible en la URL).
  • Útil para envíos de formularios donde un usuario desea marcar el resultado.
  • GET es mejor para datos no seguros, como cadenas de consulta en Google.

Utilice siempre POST si los datos del formulario contienen información confidencial o personal. El método POST no muestra los datos del formulario enviado en el campo de dirección de página.

  • POST no tiene limitaciones de tamaño y puede usarse para enviar grandes cantidades de datos.
  • Los envíos de formularios con POST no pueden marcarse.

Cada campo de entrada debe tener un name atributo para ser enviado. Si namese omite el atributo, los datos de ese campo de entrada no se enviarán en absoluto.

El fieldset elemento se usa para agrupar datos relacionados en un formulario. El legend elemento define un título para el elemento.

Atributos de formulario

accept-charset: Especifica el conjunto de caracteres utilizado en el formulario enviado (predeterminado: el conjunto de caracteres de la página).

action: Especifica una dirección (url) donde enviar el formulario (predeterminado: la página de envío).

autocomplete: Especifica si el navegador debe completar automáticamente el formulario (predeterminado: activado).

enctype: Especifica la codificación de los datos enviados (valor predeterminado: está codificado en url).

method: Especifica el método HTTP utilizado al enviar el formulario (predeterminado: GET).

name: Especifica un nombre utilizado para identificar el formulario (para uso DOM: document.forms.name).

novalidate: Especifica que el navegador no debe validar el formulario.

target: Especifica el destino de la dirección en el atributo de acción (predeterminado: _self).

El elemento select

Define una lista desplegable.

Los option elementos definen una opción que se puede seleccionar.

De manera predeterminada, se selecciona el primer elemento de la lista desplegable. Para definir una opción preseleccionada, agregue el selected atributo a la opción.

Use el sizeatributo para especificar el número de valores visibles.

Use el multiple atributo para permitir al usuario seleccionar más de un valor.

El elemento textarea

Define un campo de entrada de varias líneas (un área de texto).

El rows atributo especifica el número visible de líneas en un área de texto.

El cols atributo especifica el ancho visible de un área de texto.

También puede definir el tamaño del área de texto utilizando CSS.

El elemento button

Define un botón en el que se puede hacer clic.

El elemento datalist

Especifica una lista de opciones predefinidas para un input.

Los usuarios verán una lista desplegable de las opciones predefinidas a medida que ingresan datos.

El list atributo del input, debe referirse al id del datalist.

El elemento output

Representa el resultado de un cálculo (como el realizado por un script).

El elemento input

Estos son los diferentes tipos de entrada que puede usar en HTML:

  1. button

  2. checkbox

    3. define una casilla de verificación. Las casillas de verificación le permiten al usuario seleccionar CERO o MÁS opciones de un número limitado de opciones.

  3. color

    4. utiliza para campos de entrada que deben contener un color. Dependiendo de la compatibilidad del navegador, puede aparecer un selector de color en el campo de entrada.

  4. date

    5. se utiliza para los campos de entrada que deben contener una fecha. Dependiendo del soporte del navegador, un selector de fecha puede aparecer en el campo de entrada. También puede usar los atributos min y max para agregar restricciones a las fechas.

  5. datetime-local

    6. especifica un campo de entrada de fecha y hora, sin zona horaria. Dependiendo del soporte del navegador, un selector de fecha puede aparecer en el campo de entrada.

  6. email

    7. utiliza para campos de entrada que deben contener una dirección de correo electrónico. Dependiendo del soporte del navegador, la dirección de correo electrónico se puede validar automáticamente cuando se envía.

  7. file

  8. hidden

  9. image

  10. month

    11. permite al usuario seleccionar un mes y año. Dependiendo del soporte del navegador, un selector de fecha puede aparecer en el campo de entrada.

  11. number

    12. define un campo de entrada numérico. También puede establecer restricciones sobre qué números se aceptan.

  12. password

    13. define un campo de contraseña. Los caracteres en un campo de contraseña están enmascarados (se muestran como asteriscos o círculos).

  13. radio

    14. define un botón de radio. Los botones de opción le permiten al usuario seleccionar SOLO UNA de un número limitado de opciones.

  14. range

    15. define un control para la introducción de un número cuyo valor exacto no es importante (como un control deslizante). Rango por defecto es de 0 a 100. Sin embargo, se puede establecer restricciones en lo que los números son aceptados con el min, max y step atributos

  15. reset

    16. define un botón de reinicio que se recuperan todos los valores del formulario a sus valores por defecto. Si omite el atributo de valor del botón, este obtendrá un texto predeterminado.

  16. search

    17. se utiliza para campos de búsqueda (un campo de búsqueda se comporta como un campo de texto normal).

  17. submit

    18. define un botón para enviar datos de formulario a un controlador de formulario. Si omite el atributo de valor del botón Enviar, el botón obtendrá un texto predeterminado.

  18. tel

    19. utiliza para campos de entrada que deben contener un número de teléfono.

  19. text

    20. define un campo de entrada de texto de una línea.

  20. time

    21. permite al usuario seleccionar un tiempo (sin zona horaria). Dependiendo del soporte del navegador, un selector de tiempo puede aparecer en el campo de entrada.

  21. url

    22. usa para campos de entrada que deben contener una dirección URL. Dependiendo de la compatibilidad del navegador, el campo de la URL se puede validar automáticamente cuando se envía. Algunos teléfonos inteligentes reconocen el tipo de url y agregan ".com" al teclado para que coincida con la entrada de url.

  22. week

    23. permite al usuario seleccionar una semana y el año.

Los nuevos tipos de entrada que no son compatibles con navegadores web más antiguos se comportarán como que fueran de tipo texto.

Restricciones de entrada

  1. checked: Especifica que se debe preseleccionar un campo de entrada cuando se carga la página (para type = "checkbox" o type = "radio").
  2. disabled: Especifica que un campo de entrada debe estar deshabilitado.
  3. max: Especifica el valor máximo para un campo de entrada.
  4. maxlength: Especifica el número máximo de caracteres para un campo de entrada.
  5. min: Especifica el valor mínimo para un campo de entrada.
  6. pattern: Especifica una expresión regular para verificar el valor de entrada.
  7. readonly: Especifica que un campo de entrada es de solo lectura (no se puede cambiar).
  8. required: Especifica que se requiere un campo de entrada (debe completarse).
  9. size: Especifica el ancho (en caracteres) de un campo de entrada.
  10. step: Especifica los intervalos de números legales para un campo de entrada.
  11. value: Specifies the default value for an input field.

Atributos

value: Especifica el valor inicial para un campo de entrada.

readonly: Especifica que el campo de entrada es de solo lectura (no se puede cambiar).

disabled: Especifica que el campo de entrada está deshabilitado. Un campo de entrada deshabilitado no se puede usar ni hacer clic, y su valor no se enviará al enviar el formulario.

size: Especifica el tamaño (en caracteres) para el campo de entrada.

maxlength: Especifica la longitud máxima permitida para el campo de entrada, el campo de entrada no aceptará más que el número permitido de caracteres.

autocomplete: Especifica si un formulario o campo de entrada debe tener autocompletado activado o desactivado.

autofocus: Especifica que el campo de entrada debería enfocarse automáticamente cuando se carga la página.

form: Especifica una o más formas a las que pertenece un elemento.

formaction: Especifica la URL de un archivo que procesará el control de entrada cuando se envíe el formulario. El atributo de formación anula el atributo de acción del elemento form. El atributo de formación se usa con type="submit"y type="image".

formenctype: Especifica cómo deben codificarse los datos del formulario cuando se envían (solo para formularios con method = "post"). Anula el atributo enctype del elemento form. El formenctypeatributo se usa con type="submit"y type="image".

formmethod: Define el método HTTP para enviar datos de formulario a la URL de acción. El formmethodatributo anula el atributo de método del form elemento. El formmethod atributo se puede usar con type="submit" y type="image".

formnovalidate: Anula el atributo novalidate del elemento form. El formnovalidateatributo se puede usar con type="submit".

formtarget: Especifica un nombre o una palabra clave que indica dónde mostrar la respuesta que se recibe después de enviar el formulario. El formtarget atributo anula el atributo de destino del elemento form. El formtargetatributo se puede usar con type="submit"y type="image".

height: Especifica la altura.

width: Especifica el ancho.

list: Se refiere a un elemento datalist que contiene opciones predefinidas para un elemento input.

min y max: Especifican los valores mínimo y máximo para un elemento input. Los atributos miny maxfuncionan con los siguientes tipos de entrada: número, rango, fecha, fecha-hora local, mes, hora y semana.

multiple: Especifica que el usuario puede ingresar más de un valor en el input elemento, funciona con los siguientes tipos de entrada: correo electrónico y archivo.

pattern(regexp): Funciona con los siguientes tipos de entrada: texto, búsqueda, url, tel, correo electrónico y contraseña. Use el atributo de título global para describir el patrón para ayudar al usuario.

placeholder: Especifica una pista que describe el valor esperado de un campo de entrada (un valor de muestra o una breve descripción del formato). La pista se muestra en el campo de entrada antes de que el usuario ingrese un valor. Funciona con los siguientes tipos de entrada: texto, búsqueda, url, tel, correo electrónico y contraseña.

required: Funciona con los siguientes tipos de entrada: texto, búsqueda, url, teléfono, correo electrónico, contraseña, selectores de fecha, número, casilla de verificación, radio y archivo.

step: Especifica los intervalos de números legales para un elemento input. Ejemplo: si step = "3", los números legales podrían ser -3, 0, 3, 6, etc. El atributo de paso se puede usar junto con los atributos máximo y mínimo para crear un rango de valores legales. Funciona con los siguientes tipos de entrada: número, rango, fecha, fecha-hora local, mes, hora y semana.

novalidate: Cuando está presente, novalidate especifica que los datos del formulario no deben validarse cuando se envían. Este es un atributo de formulario.
